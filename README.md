# HITCON 2026 NFC Battle｜活動企劃書

## 一、活動定位

| 項目 | 說明 |
|------|------|
| **名稱** | HITCON 2026 NFC Battle（駭客名片遊戲） |
| **型態** | 大會期間之 **NFC 實體互動 + 行動 App**，結合名片交換、贊助商集點、積分與排行榜 |
| **核心體驗** | 會眾以 **NTAG Badge 貼紙** 或 **NTAG 卡片** 綁定大會帳號，透過手機掃描交換「數位名片」、完成攤位任務；工作人員以同一套機制 **查驗身分與領獎登記** |
| **正式活動日** | **2026/8/21（五）、8/22（六）** |

### 1.1 實體載具（兩種）

| 型式 | 說明 |
|------|------|
| **NTAG Badge 貼紙** | 貼於識別證（badge）；製作與寫入時程見「時程」。 |
| **NTAG 卡片（可印卡面）** | 獨立卡片型式，卡面由 **紀念品攤以自有設備** 輸出。會眾於 App 內以 **點陣圖編輯** 繪製預覽，至紀念品攤下單後取卡，再與帳號綁定。**App 需內建繪圖／匯出**；匯出規格（解析度、出血、色彩等）需與 **該設備** 及紀念品組內規一致。 |

### 1.2 攤位

- **基底**：工作人員識別證上之 **NTAG Badge 貼紙**（與會眾相同）。
- **加選**：可 **另外** 準備 **攤位專用 NTAG 卡片**（事前寫入）；有則集點優先掃卡片，**沒有則掃 Staff 的 NTAG Badge 貼紙**。

## 二、目標對象與主辦效益

- **會眾**：快速認識彼此、收集名片、參與贊助商互動、累積分數與排名；可選購 **客製卡面** 強化收藏與展示。
- **贊助商／攤位**：以「集點」模型促進人流；**Staff NTAG Badge 貼紙** 為預設掃描對象，**攤位 NTAG 卡片** 為額外選配（事前發放／寫入）。
- **大會主辦**：規則由系統統一紀錄，利於 **防弊、稽核、領獎資格** 一致化。
- **工作人員**：可透過 App 依 NFC 查身分與 **是否具領獎資格／是否已領取**，降低人工對帳成本。

### 2.1 事前登記 vs. 當天綁定

- **會眾與攤位**皆於 **活動前**即可取得 App，並先完成 **個人／攤位資料登錄**（暱稱、自介、頭像／點陣圖等）。
- **活動當天**重點為 **綁定實體**：將領到的 **NTAG Badge 貼紙** 或 **NTAG 卡片** 與自己的帳號在 App 內完成綁定，無須當天才填完整檔案。

## 四、玩法與現場營運

### 帳號與綁定

- 參加者以 **大會帳號** 登入 App；活動規則以 **系統紀錄** 為準，不依口頭自報身分。
- 每人將自己的 **NTAG Badge 貼紙** 或 **NTAG 卡片** 與帳號 **一對一綁定**；若該片貼紙／卡片已被他人綁定，須改換或洽工作人員，**不可多人共用同一載具**。

### 怎麼玩

- **交換名片**：兩人以手機 App 讀取對方載具，將對方納入自己的收藏／名片列表。
- **贊助商／攤位集點**：讀取贊助商或攤位指定之 NFC，累積集點進度；畫面上會依對象顯示 **會眾** 或 **贊助商／攤位** 相關資訊（例如攤位說明、集點進度）。
- **掃攤位**：該攤有發 **攤位 NTAG 卡片** 則優先掃卡片；**沒有則掃該攤工作人員識別證上的 NTAG Badge 貼紙**。
- **防弊**：須 **實際碰觸對方真實的貼紙或卡片** 完成掃描；僅傳連結或複製載具無法通過（向會眾宣導時可強調「實體碰一下」）。

### 任務與排名

- 可查看 **贊助商集點進度**（距離兌換門檻還差幾點等）。
- 設有 **大會排行榜**（依主辦訂定之計分規則）。

### 工作人員與領獎

- **Staff** 可掃參加者載具，確認身分與 **是否具領獎資格**、**是否已領過**。
- **領獎**：符合資格者於攤位或服務台由 Staff **登記領取**（例如贊助商滿額禮、排行榜名次獎等，品項以主辦公告為準）；**重複領取**由現場 SOP 與系統紀錄把關。

### 現場流程（建議順序）

1. **行前**：App 已可下載；會眾與攤位盡量 **先完成 App 內資料登錄**。
2. **載具就緒**：**NTAG Badge 貼紙**於活動前一日貼上識別證；選配之 **攤位 NTAG 卡片** 事前發到攤位並完成寫入；**紀念品客製 NTAG 卡片**依紀念品動線取卡後再行綁定。
3. **報到與綁定**：引導在 App 內將 **領到的貼紙或卡片** 與本人帳號綁定；遇「已被綁定」等狀況由工作人員協助排除。
4. **活動進行**：宣導以 **手機靠近實體載具** 完成名片交換與集點；**掃攤位**規則同「怎麼玩」一節。
5. **領獎**：依資格與 Staff 引導完成領取與登記。
6. **撤場**：主辦依需要公告活動結束後之排行榜或兌獎截止；資料保存與檢討依主辦政策辦理。

## 六、資源與合約檢核（辦活動必查）

- **身分**：與大會 SSO／KKTIX 等 **JWT 核發方** 確認 `sub`、Staff `role`、**攤位／贊助商** 身分與 JWT 之對應。
- **網域**：`game.hitcon2026.online` 之 DNS、憑證、**不可對 `/.well-known/*` 做會破壞驗證的轉址**。
- **App 商店**：`deeplink-hosting/b/index.html` 內 Android package id、App Store id 需與實際上架一致。
- **硬體**：**NTAG Badge 貼紙**與 **NTAG 卡片** 規格（可寫入 NDEF、UID 可讀）；現場準備 **Staff 用機** 與備品；**攤位 NTAG 卡片**（選配）與 **Staff NTAG Badge 貼紙**（預設）並存之 SOP；紀念品 **卡面** 由 **自有輸出設備** 製作，與 App 匯出規格事先對表。
- **法遵與個資**：顯示名稱、頭像、收集紀錄等之揭露範圍與隱私聲明，需由主辦定稿（本 repo 不涵蓋法務文件）。

## 七、風險與對策（企劃摘要）

| 風險 | 對策方向 |
|------|----------|
| 複製或偽造載具 | 系統比對 **載具與帳號**；宣導須 **實體掃描**。 |
| 現場網路或 App 安裝失敗 | 預留說明頁／人工備援流程（主辦訂定）。 |
| **NTAG Badge 貼紙**／**NTAG 卡片** 與名單對錯 | 寫入／發放時 **對照表與抽樣複驗**；活動日前再抽檢。 |
| 客製卡面輸出與 App 預覽不一致 | 以 **自有設備** 先行試印校色，鎖定 **安全區、解析度、色彩**；App 匯出 **唯一規格**。 |
| 攤位未加發 **NTAG 卡片** | 改掃 **Staff NTAG Badge 貼紙**；系統能辨識該 Staff 所屬攤位。 |
| Staff 誤登記領獎 | 以系統顯示之 **可領／已領** 為準；雙人覆核等 SOP 由主辦訂定。 |
